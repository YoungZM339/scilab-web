# 安全策略

## 支持范围

项目仅为当前主分支和最新发布版本提供安全修复。部署者应及时更新依赖、使用 HTTPS，并把管理后台限制在可信网络或 VPN 内。

## 私密报告漏洞

请使用代码托管平台提供的私密 Security Advisory/漏洞报告功能联系维护者。仓库尚未配置私密渠道时，请先联系仓库所有者索取安全联系方式。不要在公开 Issue、讨论区或社交媒体披露以下内容：

- 可绕过管理员认证、读取草稿或媒体的方法；
- XSS、CSRF、路径穿越、SQL 注入或任意文件读写；
- 密钥、数据库、备份、会话 token 或个人数据；
- 可稳定造成数据破坏或服务不可用的步骤。

报告请包含受影响版本、环境、最小复现、影响范围和建议修复。维护者应在确认收到后尽快回复，在补丁和升级窗口确定前保持协调披露。

## 部署者责任

- 必须设置至少 32 字符的随机 `BETTER_AUTH_SECRET`，不得复用示例值。
- 使用 TLS 反向代理；公网后台建议再加 VPN、IP 白名单或单点入口。
- SQLite 与上传目录只能放在 VPS 本地持久盘，同一数据库只能由一个应用实例写入。
- 每日执行一致性备份并加密异地保存，定期恢复演练。
- 不把 `.env`、`data/`、`backups/` 或 `ops/backups/` 提交到版本库或镜像。

更多加固说明见 [部署指南](docs/deployment.md) 和 [运维文档](docs/operations.md)。
